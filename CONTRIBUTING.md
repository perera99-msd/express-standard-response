# Contributing Guide

Thank you for your interest in contributing to express-standard-response.

## Ways to Contribute

- Report bugs
- Propose features
- Improve documentation
- Submit pull requests

## Development Setup

1. Fork and clone the repository.
2. Install dependencies:

```bash
npm install
```

3. Create a branch from develop:

```bash
git checkout -b feat/short-description
```

## Pull Request Checklist

- Keep changes focused and minimal.
- Write clear commit messages.
- Update documentation for behavior changes.
- Ensure tests pass locally.
- Link related issues in the PR description.

## Coding Standards

- Use consistent formatting with existing code.
- Keep modules small and readable.
- Prefer explicit, descriptive naming.

## Reporting Issues

When opening an issue, include:

- Expected behavior
- Actual behavior
- Reproduction steps
- Node.js version
- Express version

## Questions

Open a GitHub issue for support and discussion.
